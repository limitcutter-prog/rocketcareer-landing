## 2. 타겟 페르소나

> v1 유지. 발사·런칭 메타포 관점의 한 줄(🚀)만 추가.

### 페르소나 1 — 방향 없는 진입자 "나침반이 없다"

```
상황:  졸업 예정 or 사회초년생 1년차 미만
고민:  직무 선택 자체가 불확실 ("이 직무가 5년 뒤에도 있을까?")
니즈:  서류 교정이 아니라 방향 설정
트리거: "내가 선택한 방향이 맞다는 확인"
반응:  직무 가치, 커리어 경로, 산업별 현실 콘텐츠
🚀 발사 비유: "어느 궤도로 쏠지 정하지 못한 단계"
```

### 페르소나 2 — 막힌 3~5년차 "열심히 했는데 어디쯤인가"

```
상황:  첫 이직 or 두 번째 이직 고민 중
고민:  "이직해야 하나, 더 있어야 하나" / "내 연봉이 시장 대비 적정한가"
니즈:  시장 위치 객관화 + 이동 방향 판단
트리거: "내 서류가 심사관 눈에 어떻게 보이는가"
반응:  연차별 현실, 이직 타이밍, 연봉 구조 콘텐츠
🚀 발사 비유: "발사대에 섰는데 점검 리포트가 필요한 단계"
```

### 페르소나 3 — 결단이 필요한 7~10년차 "이번이 마지막 기회일 수 있다"

```
상황:  관리자 트랙 or 전문가 트랙 기로
고민:  "이 경력이 밖에서 얼마나 먹히는가"
니즈:  경력의 시장가치 직접 판단
트리거: "HR담당자 눈에 내가 어떻게 보이는가"
반응:  고연차 이직 현실, 직급별 시장 수요 콘텐츠
🚀 발사 비유: "한 번의 발사 실패가 비싼, 정밀 점검이 필요한 단계"
```

### 마스터 메타포 시스템 (IMC 일관성 장치)

> 모든 채널·콘텐츠·퍼널이 같은 메타포를 공유하면 인지가 누적된다.

| 발사 메타포 | 실제 의미 | 적용 지점 |
|---|---|---|
| 🚀 **발사 / 런칭** | 합격·이직 성공, 커리어의 도약 | 브랜드명·인트로·CTA |
| 🛰️ **발사 전 점검** | 무료 진단 (커리어 런칭 세션) | 진입 상품·리포트 |
| 🗼 **관제탑 / 비행 엔지니어** | 현직 HR멘토그룹 | 차별화 메시지 |
| 🛫 **발사대** | 로켓커리어연구소 플랫폼 | 랜딩·서비스 |

### 캐치프레이즈 (v2 확정)

```
[마스터 캐치 — 브랜드 약속]
당신의 커리어, 발사 준비 완료.

[디스크립터 — 무엇을 하는 곳인가]

[문제제기 레이어 — 캠페인/광고 카피로 보존]
```

**구조 분석:**
- 마스터 캐치는 **결과·도약(긍정)**, 문제제기 레이어는 **결핍·위험(공감)** — 둘을 캠페인 목적에 따라 조합
- "런칭"은 서비스 브랜드(커리어런칭 서비스)·진입 상품(커리어 런칭 세션)과 어휘가 일치 → 메시지-상품 정합

### 콘텐츠 화법 원칙 (v2 — sync-arch 화법 룰 정합)

> 콘텐츠(영상 대본·자막·고정댓글)에 적용되는 강제 룰. 코드 SoT는 `lib/marketing/casefile-script.ts` SYSTEM_PROMPT. (WORKLOG 2026-06-03 "훅·나레이션 IMC 화법 승인" + 2026-06-01 "훅 강화")

```
❌ 금지 패턴 (hard rule):
  "자소서 잘 쓰는 법 5가지" 류         → AI·코치와 구분 안 됨
  "이렇게 하면 합격합니다"             → 근거 없는 보장
  "HR담당자가 말합니다" (HR이 주어)     → 익명성·시점 위반
  "다음편은 ~~편" (시리즈 예고)        → 회차 유동적, 신뢰 리스크
  첨삭 직접 권유 ("첨삭 받으세요")       → 진단/점검 프레이밍으로 대체

✅ 하는 것:
  시점:   HR 내부 → 지원자에게 전달 (주어 = 상황·현상)
  목소리: 집단의 목소리 ("저희가 수백 건 검토하면서 공통적으로…")
  진실:   불편하지만 유용한 진실 (AI는 듣기 좋게 최적화, HR은 "이거 걸린다"고 직접)
  구체:   "수치를 넣으세요"(X) → "수치 없는 항목 나오면 HR이 하는 생각"(O)

✅ 톤앤매너 (memory: feedback_tone_voice):
  합니다체 + 해요체 혼용
  - 훅/오프닝: 해요체 ("~하시는 것 같아요", "~었어요")
  - 사례 제시: 합니다체 ("~있었습니다", "~없습니다")
  - CTA:      해요체 ("~해보세요", "~있으세요?")
  방법론 이름 금지 → 행위 중심 ("GAP 분석" X → "낭비되는 구간을 찾아" O)
```
