# /casefile-qa — CaseFileVideo 대본 자동 검수

`/casefile-script` 출력(props + ttsScenes)을 3관점으로 검수한다. 불량은 재작성 피드백을 준다.
영상 렌더 전 **불량 사전 차단**이 목적.

## 입력
- CaseFileVideoProps + ttsScenes (또는 완성된 영상 대본)

## 검수 3관점

### 관점 1 — 논리·사실 정합성 (가장 치명적)
- [ ] 모순 없는가? (예: "자소서 통과인데 서류 탈락" = 모순)
- [ ] 인과/시점 어긋남 없는가?
- [ ] before→after 변환이 실제로 개선인가? (숫자·행동이 추가됐는가)

### 관점 2 — HR 언어·톤앤매너
- [ ] 단계=`서류전형`, 문서=`경력기술서/이력서` 정확히 구분했는가?
- [ ] 비-HR 표현 없는가? ("막혔다/버렸다" 류)
- [ ] 훅=해요체 공감 / 사례=합니다체 / CTA=해요체 온기, 혼용 맞는가?
- [ ] 단정형 과다 아닌가? (공감·온기 살아있는가)

### 관점 3 — 취준생 몰입·구조
- [ ] 훅이 첫 2초에 공감 페인포인트를 찌르는가?
- [ ] 의뢰인 파일에서 "문제가 뭔지" 명확한가?
- [ ] pair마다 "왜 약한지"가 구체 설명됐는가? (단정만 ❌)
- [ ] 자막(diff·closing) 한 줄인가?
- [ ] CTA에 저장+댓글창링크 2요소 다 있는가?
- [ ] CTA 결론이 **커리어 진단/점검·고민** 프레이밍인가? ("첨삭해드려요" 류 직접 권유면 fail — 첨삭 직접 언급 지양)
- [ ] CTA가 **짧은가?** (한 줄 ≤18자·총 2~3줄. 길면 자막 줄 깨짐 → fail)
- [ ] ⚠️ CTA에 **시리즈 예고("다음편은 ~~편")가 없는가?** (있으면 fail — 회차 유동적, 2026-05-30)

### TTS 체크
- [ ] 문장 끝 하강조 평서형 통일?
- [ ] 과한 쉼표로 어색한 끊김 유발 안 하는가?

## 출력 형식
```
[casefile-qa 결과]
✅ 통과: [관점/항목]
⚠️  수정필요: [항목] — [사유] → [수정 제안]
🔴 치명적: [항목] — [반드시 재작성]

종합: 통과 / 수정 후 통과 / 재작성 필요
```

## 루프
🔴 또는 ⚠️ 2개 이상이면 → `/casefile-script`로 해당 부분 재작성 → 재검수.
