# /casefile-script — 케이스 → CaseFileVideo 대본 생성

`cases_career` 레코드(또는 사용자가 준 케이스 정보) 1건을 받아, CaseFileVideo(v7) 영상용
**props + TTS 나레이션 대본**을 생성한다. 품질의 80%를 결정하는 핵심 스킬.

> 전제: `admin-tool/remotion/CLAUDE.md`의 "콘텐츠 작성 룰 10개"가 최우선 기준. 아래는 그 운영 절차.

## 입력
- 케이스: 직무/연차/산업/상황 + before/after 사례 2~4개 (cases_career 또는 자유 입력)
- (옵션) 목소리: 기본 `google / ko-KR-Chirp3-HD-Zubenelgenubi / rate 1.2`

## 출력 (3개 블록)
0. **hook_candidates[]** (훅 후보 2~3개 — 서로 다른 각도, 어드민에서 선택)
1. **CaseFileVideoProps** (화면 자막용 — 키워드/한 줄)
2. **ttsScenes[]** (나레이션용 — 완결 문장, 자연 구어)

## 트렌드 벤치마크 힌트 (옵션)
- `trend_benchmarks`(①이 쌓은 9항목 분석)에서 잘 먹히는 훅 패턴을 "[참고 트렌드]" 블록으로 주입(`lib/marketing/casefile-benchmark.ts`).
- ⚠️ **참고용일 뿐** — 룰 1~13보다 우선하지 않음. 억지로 끼우지 않음. 없으면 폴백.

```ts
// 1) 화면 props
{
  case_number, hook,                         // hook = 공감형 한 줄(자막)
  context: { job_role, career_level, industry?, situation },  // situation = 문제 명확
  pairs: [{ before, after, diff }...],       // diff = 한 줄 핵심(왜는 음성이 설명)
  closing,                                    // 날카로운 결론 한 줄
  cta                                         // 댓글창 유도 + 저장
}
// 2) 나레이션 (씬별 완결 문장)
[{ scene:'hook'|'context'|'pair_0'..|'closing'|'cta', text }]
```

## 작성 룰 (필수 — 위반 시 재작성)

### 논리·언어
1. **논리 정합**: "자소서 통과인데 서류 탈락" 류 모순 금지. (자소서=서류의 일부)
2. **HR 언어**: 단계는 `서류전형`, 문서는 `경력기술서/이력서`. "경력기술서에서 탈락"❌ → "서류전형에서 탈락"✅. "걸러졌다/문턱을 넘지 못했다" 사용. 검토 주체는 **"채용담당자/HR"** 통일, "심사관"❌.
2b. **이력서체(필수)**: pairs.**after**(개선문)는 **명사형·개조식**. "자동화해요"❌→"프로세스 자동화로 처리시간 단축"✅. (before는 의뢰인 실제 약점 원문 → 원본 그대로). 해요체/합니다체는 나레이션·hook·cta에서만.
2c. **가독성(필수)**: after·diff는 일반 취준생도 한 번에 이해되게. 전문 툴명·약어·내부용어(BlackLine, Rollover, Reconcile, Swap…)는 **쉬운 한국어로 풀어** 쓴다(수치·성과는 유지). 예: "BlackLine Reconcile 74%→98%"→"결산 대사 정확도 74%→98%". diff는 어려운 비유 금지, 평이한 한 문장으로 "왜 더 나은지" 즉시 납득. 못 알아듣는 after는 실패.

### 톤앤매너 (memory: feedback_tone_voice / project_shorts_production)
3. 진중한 인사담당자 + 체험기반·공감·온기. 합니다체+해요체 혼용.
   - **훅 = 해요체 공감형**: 취준생 페인포인트 단어 ("열심히 썼는데 왜 떨어질까요", "~것 같으세요?")
   - **사례 = 합니다체**: "~보이지 않습니다", "신뢰가 갑니다"
   - **CTA = 해요체 온기**: "~확인해보세요", "도움이 될 거예요"

### 구조·설명 (차별점)
4. **왜를 설명**: "이건 탈락입니다" 단정 금지. 인사담당자가 *왜* 그렇게 보는지 구체 설명(길어도 OK).
   - pair 나레이션 구조: ①원문 지목 → ②왜 약한지(HR 관점) → ③어떻게 바꾸나 → ④왜 통하나
5. **자연 쿠션어**: 작위적("첫 문장입니다")❌ → 지목형("가장 먼저 눈에 걸린 문장입니다", "이 문장도 마찬가지입니다", "가장 흔한 실수입니다")✅
6. **자막은 한 줄**: pairs.diff·closing은 한 줄 핵심만. "왜"는 나레이션이 풀어서.
7. **결론**: 그 영상 실제 인사이트 요약, 날카롭고 명확하게.

### CTA (절대 누락 금지)
8. 매 영상 ① 저장 유도 ② **댓글창 링크**(소프트 유도, 서비스명 직접 홍보 금지) 포함.
   - **결론은 "첨삭" 직접 언급 대신 커리어 진단/점검·고민으로 마무리** ("첨삭해드려요"❌ → "커리어 한 번 점검해보세요"✅). 인사담당자가 주는 명확성 유지.
   - **짧게(엄수)**: 한 줄 ≤18자·총 2~3줄(길면 자막 줄 깨짐). 아래 3톤 중 케이스에 맞는 1개 변주(매번 동일 금지):
     (a) 점검 권유 "내 경력기술서는 어떤 인상일지\n한 번 점검해보세요"
     (b) 호기심 자가진단 "내 서류는 채용담당자 눈에\n어떻게 보일까요?"
     (c) 공감 셀프체크 "혹시 내 서류도 같은 이유로\n걸러지고 있는 건 아닐까요?"
   - ⚠️ **시리즈 예고("다음편은 ~~편") 금지** — 회차 유동적이라 약속 못함. "저장해두고 다시 보세요" 류로 대체. (2026-05-30)

### TTS 자연성
9. 문장 끝 하강조 평서형(~습니다/~다) 통일(rate 1.2 말꼬리 상승 방지). **과한 쉼표 금지**(Chirp3-HD 어색한 gap). 연결어(자/정작/그제야/이렇게) 활용.

### 길이 (자수 예산 — 엄수)
10. hook ≤45자 / context ≤70자 / closing ≤55자 / cta ≤80자 / **각 pair 나레이션 ≤70자·최대 4문장(절대 85자 초과 금지)**. 전체 45~70초. ("길게"❌ "정확하게"✅)

### 추가
11. **맞춤법**: 출력 전 오탈자·비문 점검("주어면→주어지면","써여야→써야"). 어색한 조사/어미 교정.
12. **모드(mode)**: `universal`=직군명·전문용어를 일반 직장인 언어로 치환(도달↑). `specific`(기본)=직군 통용어 허용+쉬운 설명 유지.

## 실행 절차
1. 케이스에서 직무·연차·산업·핵심 문제 추출 → `context` 구성 (situation에 문제의식 명확히)
2. before/after 사례 2~4개 선별 → 각 pair의 자막(한 줄)과 나레이션(왜 설명) 작성
3. 공감 훅(자막) + 진행자 개입형 hook 나레이션 작성
4. 날카로운 closing + 댓글창 CTA 작성
5. 2블록(props / ttsScenes) 출력 → 이어서 `/casefile-qa`로 검수 권장

## 다음 단계 연결
출력한 ttsScenes → `scripts/regen-tts-*.mjs` 패턴으로 TTS 생성(공백제거 포함) → 실측 duration을
props.pairs[].duration / timing에 반영 → Root.tsx 또는 DB 저장 → 렌더.
상세 파이프라인: `admin-tool/CASEFILE_PIPELINE.md`
