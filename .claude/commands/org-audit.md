# /org-audit — 객관성 검증 (아첨·거짓보고 차단)

> `org_orders`(+ 동결 히스토리 `LEDGER.md`)의 완료/검증 주장을 **실제 근거와 대조**한다. 운영체계의 "거짓말 금지"를 강제하는 게이트.
> `casefile-qa`(실행≠검수)의 조직 버전. `CHARTER.md` §1·§2 적용.

## 실행 절차

1. `cd admin-tool && npm run orders -- list 완료`(+`list 검증`)로 항목 추출. `result.evidence`·`org_order_events` 근거 확인. (과거분은 `org/LEDGER.md` 참고)
2. 각 항목의 **인용 근거를 실제로 열어 대조**:
   - `파일경로:줄` → 존재·내용 확인
   - 지표값 → 재계산/쿼리 일치 확인
   - 커밋해시 → `git log` 존재 확인
3. 근거 없음·불일치 → `npm run orders -- update <id> --status 반려 --note "사유"`로 강등.
4. **증거 등급 위반 점검**(CHARTER §2): 조회수로 성과 주장, S5 없이 합격률 외부 사용 등 → 플래그.
5. 리포트.

## 출력 형식
```
[org-audit 결과]
✅ 검증 통과: N건
⚠️ 반려: [ID] — [사유: 근거 없음/불일치]
🚫 증거등급 위반: [ID] — [예: 조회수로 성과 주장]
마지막 감사: YYYY-MM-DD
```

> 원칙: 의심스러우면 통과시키지 않는다. "됐다"는 주장은 재현 가능한 근거가 있어야만 `검증`으로 유지.
